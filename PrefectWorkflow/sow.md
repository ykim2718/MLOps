# Statement of Work (작업 명세서)

| Field | Content |
|---|---|
| Project | 다중 사용자 AI/ML modeling·실험 이력관리 플랫폼 구축 |
| Basis | Senior Engineer 1인 전담 |
| Approach | 사양서 미제공 (Greenfield) — 요구 한 줄에서 설계부터 수행 |
| Duration | 5~6개월 (견고화 포함 시 6~7개월) |
| Date | 2026-06-21 |

## 1. 개요 (Overview)

  팀원 다수가 동시에 AI/ML modeling 작업을 돌리고 그 이력을 추적·재현하며 best trial 을 검색하는 self-hosted MLOps 플랫폼을 구축합니다. 요구가 한 줄 수준이라 사양·아키텍처를 처음부터 설계하며, 본 명세서는 구현뿐 아니라 **요구 구체화·기술 선정·설계** 를 포함합니다.

## 2. VOC (Voice of Customer)

  발주처 원문 요구는 한 줄입니다.

  > "다수 팀원이 동시다발로 AI/ML modeling 작업을 수행하고, 각 실행 이력을 관리·재현하며 best trial 을 검색할 수 있어야 한다."

  이를 충족할 기술 요구 (모두 설계 대상):

  - **동시 실행·격리** — 팀원별 job 동시 실행 + 자원 등급 (고·저) 분리
  - **재현성** — 실행마다 코드 (git commit)·데이터·환경 버전 고정 → 같은 입력에 같은 결과
  - **이력·계보 (lineage)** — 실행자·시각·코드·파라미터·결과 (메트릭·아티팩트) 추적
  - **Best trial 검색** — 지표 기준 trial 랭킹·검색 (top-N, 팀원·기간 필터) + Optuna study 연동
  - **저장·운영** — 데이터·모델·아티팩트 저장소, 백엔드 DB, 자격증명·네트워크·기동 절차

## 3. 범위 (Scope)

### 3.1 In Scope

  - 요구 구체화 및 아키텍처 설계 (기술 선정 PoC 포함)
  - 오케스트레이션 플랫폼 구축 (동시 실행·자원 등급·job 격리)
  - 실행 단위 코드·데이터·환경 버전 고정 오케스트레이터
  - 재현성 보장·검증 (결정적 재실행)
  - 실험 추적·이력 관리 연동
  - Best trial 검색·랭킹 (지표 기준 top-N, Optuna study 연동)
  - 객체 저장소·백엔드 DB·시크릿·네트워크 구성
  - 동시성 부하 시험 및 안정화
  - 운영 문서·인수인계

### 3.2 Out of Scope

  - AI/ML 모델·알고리즘 자체의 개발 (팀원 코드는 발주처 책임)
  - 실데이터 대량 이행 (별도 협의)
  - 클라우드/온프렘 운영 인프라 조달 및 비용
  - 24/7 운영·모니터링·장애 대응 (SLA 별도 계약)

## 6. 기간 (Timeline)

  Senior 1인 전담 기준 **총 5~6개월**, 견고함·운영 편의까지면 **6~7개월** 입니다.

| Milestone | Cumulative |
|---|---|
| M1 설계 확정 | ~4주 |
| M2 코어 동작 (단일 사용자 end-to-end) | ~12주 |
| M3 동시 실행·재현·검색 안정화 | ~18주 |
| M4 인수인계 완료 | ~22주 |

  1개월이 아닌 까닭:

  - 설계 단계 신규 추가
  - 모호한 요구 → 반복·재작업
  - 재현성 보장 — 결정적 재실행 검증 추가
  - best trial 검색 — 동시 trial 간 지표 집계·랭킹 계층 추가
  - "돌아간다" ↔ "동시에 돌려도 안 깨진다" 간극 메우는 안정화 별도

## 10. 검수 기준 (Acceptance Criteria)

  - 동시 실행 시 결과·자원 상호 침범 없음
  - 동일 코드·데이터·환경으로 결정적 재현 (같은 입력 → 같은 결과)
  - 실행별 이력 (실행자·시각·코드·파라미터·결과) 조회 가능
  - 지표 기준 best trial 검색·조회 가능 (top-N 랭킹, 팀원·기간 필터)
  - 실패 실행이 원인과 함께 이력에 기록
  - 운영 문서만으로 신규 인원이 기동·실행 가능
